## Problem

`/clawdbrt:record-session` (and `python -m clawdibrate --dump-session`) currently only reads Claude Code session JSONL files from `~/.claude/projects/<mangled-path>/`. This makes the feature Claude-only — it won't work for users running Codex, OpenCode, Cursor, or other agent CLIs.

## Deliverable

Make `session_dump.py` agent-agnostic:

1. **Detect which agent produced the session.** Check for known session storage paths:
   - Claude Code: `~/.claude/projects/<mangled-cwd>/*.jsonl`
   - Codex: TBD — investigate session storage location
   - Cursor: TBD — investigate `.cursor/` or similar
   - OpenCode: TBD — investigate session storage

2. **Pluggable parsers.** Refactor `dump_session()` to dispatch to an agent-specific parser based on the detected format. Each parser converts the agent's native session format into clawdibrate's canonical transcript JSONL (same `event`, `tool_call`, `user_message`, `assistant_message` schema).

3. **`--agent` flag for `--dump-session`.** Allow explicit agent selection:
   ```bash
   python -m clawdibrate --dump-session --agent codex
   ```

4. **Fallback.** If the agent's session format is unknown or not stored locally, print a clear error suggesting `/clawdbrt:record-start` as the universal alternative.
