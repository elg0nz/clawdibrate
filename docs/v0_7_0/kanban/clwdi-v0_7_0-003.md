## Problem

The calibration loop optimizes AGENTS.md for the specific transcripts it's seen, risking degradation on unseen sessions. No mechanism exists to detect overfitting.

## Deliverable

Implement train/test transcript splitting in the orchestrator:

1. **Split transcripts.** 80/20 split by default. For small counts (<5), use leave-one-out. New flag: `--holdout-ratio 0.2`.
2. **Calibrate on train set only.** The bug-identifier, judge, and implementer only see train transcripts.
3. **Score on test set.** After implementing changes, run deterministic metrics on held-out transcripts without modifying AGENTS.md.
4. **Detect overfitting.** If test score degrades while train score improves, revert the change and log a warning. Persist train/test scores separately in `.clawdibrate/history/scores.jsonl`.
5. **Report.** Print train vs test scores in the calibration summary.
