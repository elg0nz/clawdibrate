## Problem

Old transcripts can drive changes that hurt recent sessions. A fix optimized for a 3-month-old transcript may degrade current behavior.

## Deliverable

Weight transcripts by recency:

1. **Decay function.** Exponential decay based on transcript age. Recent sessions get weight 1.0, older sessions decay toward a floor (e.g., 0.3).
2. **Apply to scoring.** Multiply failure scores by the transcript's recency weight before aggregation.
3. **Conflict resolution.** When an old transcript and a recent one suggest conflicting fixes, the recent one wins.
4. **Configurable half-life.** `--staleness-halflife-days 30` (default 30 days).
