## Problem

`npx skills add ./src/skills --all -y` installs skills to all 45 known agents regardless of which ones are actually present. This is noisy (overwrites warnings for agents that don't exist) and installs to directories that will never be used.

## Deliverable

Replace `--all` with agent-aware installation:

1. **Detect installed agents.** Check which agent CLIs are available (e.g., `which claude`, `which codex`, `which cursor`, etc.) and which have config directories present (`.claude/`, `.cursor/`, `.codex/`, etc.).

2. **Install only to detected agents.** Pass specific agent names to `npx skills add` instead of `--all`. The `skills` CLI supports targeting specific agents — use that instead of the blanket flag.

3. **Update all callsites.** The `--all` pattern appears in:
   - `clawdibrate/instruction_files.py` (`_install_bundled_skills`)
   - `AGENTS.md` (Setup section, Boundaries section)
   - Skill files that reference the install command

4. **Fallback.** If no agents are detected, warn and skip installation (don't error). Suggest manual install with the specific agent flag.
