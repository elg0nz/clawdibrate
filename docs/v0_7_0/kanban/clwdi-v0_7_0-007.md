## Problem

The first real calibration run (2026-04-02) leaked implementer prompt artifacts into AGENTS.md:
- "Here is the updated section body:" prepended to the Boundaries section
- "Summary of changes:" block with rationale appended after Boundaries

These are meta-commentary from the implementer LLM that `replace_section()` inserted verbatim.

## Fix

1. **Post-processing in orchestrator.** After the implementer returns new section content, strip common artifact patterns before calling `replace_section()`:
   - Lines matching `^(Here is|Summary of|Updated section|The following|I've)` at the start of the content
   - Trailing `**Summary**` or `**Changes**` blocks
2. **Prompt hardening.** Update `prompts/implementer.md` to explicitly forbid meta-commentary: "Output ONLY the replacement section body. No preamble, no summary, no explanation."
3. **Validation.** After replacement, verify the section doesn't contain obvious prompt leak patterns. If detected, reject and log.
