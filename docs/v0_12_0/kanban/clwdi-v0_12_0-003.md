# clwdi-v0_12_0-003 — Version bump should be a skill

**Status:** todo
**Priority:** medium
**Complexity:** S

## Observation (from user)

Version bumping (pyproject.toml, AGENTS.md header, CHANGELOG.md) is done manually every release. By now this should be a `/clawdbrt:bump-version` skill that handles all three files atomically.

## What

Create `src/skills/bump-version/SKILL.md` that:
- Reads current version from `pyproject.toml`
- Computes the next MAJOR/MINOR/PATCH increment
- Updates `pyproject.toml`, `AGENTS.md` header, `docs/CHANGELOG.md` in one pass
- Prints a diff summary

Register via `npx skills add ./src/skills --agent claude-code cursor codex --skill '*' -y --global`.

## Acceptance

- `/clawdbrt:bump-version minor` bumps 0.12.0 → 0.13.0 in all three files
- No manual edits needed after running the skill
