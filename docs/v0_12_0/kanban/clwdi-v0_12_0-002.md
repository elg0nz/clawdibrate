# clwdi-v0_12_0-002 — Judge: penalize skill invocation failures harder

**Status:** todo
**Priority:** high
**Complexity:** S

## Observation

With sonnet on low effort, the judge appears to let skill-invocation failures slide — agents skip invoking the correct skill (e.g. `/clawdbrt:loop`) and do the work ad-hoc instead. The judge does not flag this as a meaningful failure.

## What

Tighten `clawdibrate/prompts/judge.md` to treat skill bypass as a `boundary_violation` with `fixability: high` and push `weight` toward 1.0. Specifically:

- Add an explicit rule: if the transcript shows the agent reimplemented logic that a named skill covers (e.g. manually wrote calibration steps instead of invoking `/clawdbrt:loop`), classify as `boundary_violation`, `fixability: high`.
- Note that low-effort / fast-mode models are more likely to skip skills — the judge should not give a pass just because the output was correct.

## Acceptance

- A transcript where the agent bypasses a skill gets `is_agents_md_failure: true`, `fixability: high`, `weight >= 0.7`
- No change to other verdict logic
