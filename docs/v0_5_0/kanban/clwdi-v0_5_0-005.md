## Problem

The semver line in the header says "MAJOR = loop contract breaks, MINOR = new sections/rules, PATCH = wording fixes" but doesn't tie this to skill commands. Agents may bump the wrong level.

## Expected Behavior

AGENTS.md should explicitly state:
- `/loop` → PATCH only
- `/add-new-features` → MINOR only
- MAJOR → human decision only, never auto-bumped

## Fix

Add to Boundaries:
- "✅ Always: `/loop` bumps PATCH only. `/add-new-features` bumps MINOR only."
- "🚫 Never: auto-bump MAJOR — requires explicit human decision"
