## Problem

All three skills in `skills/` use bare names (`loop`, `kanban`, `add-new-features`), which collide with other skill providers (e.g., gstack's `/loop`, `/kanban`). The system-reminder already shows both `loop` (gstack) and `loop` (clawdibrate) — users and the skill router can't distinguish them.

## Scope

Three skills need the `clawdbrt:` prefix:

| Current name        | New name                      | File                             |
|---------------------|-------------------------------|----------------------------------|
| `loop`              | `clawdbrt:loop`               | `skills/loop/SKILL.md`           |
| `kanban`            | `clawdbrt:kanban`             | `skills/kanban/SKILL.md`         |
| `add-new-features`  | `clawdbrt:add-new-features`   | `skills/add-new-features/SKILL.md` |

## Changes Required

1. **SKILL.md frontmatter** — Update `name:` field in each skill's SKILL.md to include the `clawdbrt:` prefix.
2. **AGENTS.md** — Update any skill references from bare names to prefixed names.
3. **Cross-references** — Grep the codebase for bare skill invocations (`/loop`, `/kanban`, `/add-new-features`) in docs, specs, and scripts; update to `clawdbrt:` form.
4. **Skill router contract** — If `docs/v0_3_0/skill-router-draft.md` or similar defines a NAME constant, update the example.
5. **Verify** — After renaming, confirm `/clawdbrt:loop`, `/clawdbrt:kanban`, `/clawdbrt:add-new-features` appear correctly in the system-reminder skill list and don't collide with other providers.

## Acceptance Criteria

- All three SKILL.md files use `clawdbrt:` prefix in `name:` field
- No bare-name references remain in AGENTS.md or docs
- Skills are invocable as `/clawdbrt:loop`, `/clawdbrt:kanban`, `/clawdbrt:add-new-features`
