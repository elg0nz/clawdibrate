## Problem

Agents edit files in `skills/` or `.agents/skills/` directly, which are gitignored install targets. The canonical source is `src/skills/`. This broke a Codex run in v0.4.1 when `npx skills add` overwrote tracked files.

## Expected Behavior

AGENTS.md should make clear that `src/skills/` is source of truth and `skills/` / `.agents/` are install outputs.

## Fix

Add to Skills section or Boundaries:
- "✅ Always: create/edit skills in `src/skills/{name}/SKILL.md`, then run `npx skills add ./src/skills --all -y`"
- "🚫 Never: edit files in `skills/`, `.agents/`, or agent-specific skill dirs directly"
