## Problem

AGENTS.md referenced `docs/v0_0_0/specs/agents-proto.md` directly, causing agents to use outdated specs instead of checking the latest version first. Partially fixed in v0.4.2 but the general rule needs to be stronger.

## Expected Behavior

No hardcoded version paths anywhere in AGENTS.md. All references should use "latest `docs/vX_Y_Z/`" language or explicit resolution order (latest first, fallback to older).

## Fix

- Audit all path references in AGENTS.md for hardcoded version numbers
- Replace with "latest version directory" language
- The Boundaries rule added in v0.4.2 covers runtime behavior; this card covers the static references
