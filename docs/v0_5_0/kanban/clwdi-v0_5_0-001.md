## Problem

AGENTS.md has no guidance on spawning parallel agents for independent work. The user strongly prefers parallel execution but agents default to sequential unless told otherwise.

## Expected Behavior

AGENTS.md should instruct agents to spawn parallel sub-agents for independent tasks (e.g., implementing separate kanban cards).

## Fix

Add a rule to the Boundaries or Tuning Rules section:
- "When multiple kanban cards are independent, spawn parallel agents — do not work them sequentially."
