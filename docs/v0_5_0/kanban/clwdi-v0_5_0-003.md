## Problem

Agents create kanban cards with inconsistent naming. The convention is `clwdi-v{MAJOR}_{MINOR}_{PATCH}-{NNN}.md` but this isn't documented in AGENTS.md.

## Expected Behavior

AGENTS.md should specify the ticket naming convention, icebox carry-forward rule, and rename-on-move behavior.

## Fix

Add to the Skills or Boundaries section:
- Ticket filenames: `clwdi-v{MAJOR}_{MINOR}_{PATCH}-{NNN}.md`
- Moving a ticket renames it to the target version prefix
- Each new version's kanban copies the prior version's `icebox.md`
