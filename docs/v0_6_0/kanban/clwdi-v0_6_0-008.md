## Problem

`reflection_history` is in-memory only. A crash loses all prior reflections. Violates the Ralph Loop principle: state lives in files, not context window.

## Fix

In the new architecture, the bug-identifier and judge outputs are the reflections. Persist each calibration run's full output (identified failures, scores, implemented changes) to `.clawdibrate/history/reflections.jsonl`. The implementer prompt receives recent history so it doesn't repeat past mistakes. Load on startup.
