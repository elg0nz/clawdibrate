## Problem

Without a baseline, scores are uninterpretable. A 0.90 means nothing if an empty AGENTS.md also scores 0.90.

## Fix

On first calibration run, the orchestrator should:
1. Run the same transcript's tasks with an empty system prompt (no AGENTS.md) → score as baseline
2. Run with the original AGENTS.md (before any mutations) → score as static baseline
3. Report delta-over-baseline, not raw score

Store baselines in `.clawdibrate/history/baselines.jsonl`. The bug-identifier prompt should reference baseline scores when assessing whether a failure is an AGENTS.md problem or an inherent LLM limitation.
