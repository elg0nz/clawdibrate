## Problem

The new architecture needs real transcripts. Users need a way to capture them without manual effort.

## Deliverable

Two skills in `src/skills/`:

- `/clawdbrt:record-start` — tells the agent to log all tool calls, searches, decisions, and user corrections to `.clawdibrate/transcripts/{timestamp}.jsonl` as structured events
- `/clawdbrt:record-stop` — stops recording, finalizes the transcript file, reports summary (tool calls, searches, corrections)

Alternative for repos that don't install skills: a single line added to AGENTS.md that enables the same logging behavior natively.

Transcript format should capture:
- Tool name, arguments, result summary
- Whether the tool call was a search (Glob, Grep, Read) vs. an action (Edit, Write, Bash)
- User corrections ("no not that", "use X instead")
- Time between user message and first action (hesitation = AGENTS.md gap)
