## Problem

The old loop used the same agent as both executor and judge — circular self-evaluation. Even with CLI-based invocation, using one prompt for everything conflates roles.

## Fix

Three separate prompt files, each with a distinct role and output contract:

1. `prompts/bug-identifier.md` — reads transcript + current AGENTS.md, outputs failure list
2. `prompts/judge.md` — reads failures, scores each 0.0–1.0 with rubrics
3. `prompts/implementer.md` — reads scored failures + current section, outputs rewritten section

Each is invoked as a separate CLI call with its prompt file as the system prompt. The orchestrator passes data between them but adds no intelligence. This separates concerns even though the same underlying model may serve all three — the prompts enforce different evaluation frames.
