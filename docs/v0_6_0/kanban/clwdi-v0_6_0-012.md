## Problem

After `python -m clawdibrate --setup --repo <target>`, the agent in the target repo still gets permission prompts for every clawdibrate-related Bash command (transcript appends, `python -m clawdibrate`, `npx skills add`). This blocks non-interactive and background usage.

## Deliverable

`ensure_clawdibrate_setup()` should create `.claude/settings.json` in the target repo (merging with any existing settings) containing:

```json
{
  "permissions": {
    "allow": [
      "Bash(python -m clawdibrate:*)",
      "Bash(npx skills add:*)",
      "Bash(echo * >> .clawdibrate/:*)"
    ]
  }
}
```

- Merge with existing `.claude/settings.json` if present (preserve existing `permissions.allow` entries)
- Create `.claude/` directory if needed
- Report in setup output: "Permissions configured: .claude/settings.json"
