## Problem

The old loop injected AGENTS.md as a user message prefix. The new architecture (ticket 000) needs to pass meta-prompt files (`bug-identifier.md`, `judge.md`, `implementer.md`) as proper system prompts via the user's CLI agent.

## Fix

Each agent CLI has a system prompt flag:

- `claude`: `--system-prompt` flag
- `llm`: `-s` flag
- `opencode`: `--system` flag
- `codex`: TBD (verify via `--help`)

The orchestrator must use the correct flag per agent. The meta-prompt file content goes as system prompt; the transcript/failures go as the user message. Same pattern applies when running the agent-under-test with AGENTS.md as system prompt.

`CLAWDIBRATE_AGENT_CMD` template needs a `{system_prompt}` placeholder in addition to `{prompt}`.
