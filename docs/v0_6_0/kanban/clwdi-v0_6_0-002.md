## Problem

The old loop used synthetic tasks that don't test AGENTS.md utility. Real transcripts are the ground truth. Token waste is the primary cost — every unnecessary search, clarifying question, or wrong-tool call is money burned.

## Deliverable

Create `clawdibrate/prompts/bug-identifier.md` — a meta-prompt that reads a real agent transcript and identifies AGENTS.md failures.

### Failure taxonomy (aligned with AgentBench execution outcomes)

| Category | AgentBench analog | What to detect | Token cost signal |
|---|---|---|---|
| **Unnecessary search** | Task Limit Exceeded | Agent searched for something AGENTS.md should have told it (file paths, naming conventions, tool flags) | High — each Glob/Grep/Read is ~500-2000 tokens |
| **Wrong tool/command** | Invalid Action | Agent used the wrong tool or CLI flag when AGENTS.md specifies the correct one | Medium — wasted call + retry |
| **Boundary violation** | Invalid Format | Agent violated an explicit boundary rule (e.g., used TaskCreate when forbidden) | Low token cost but high correctness cost |
| **Unnecessary clarification** | — | Agent asked the user something AGENTS.md already answers | Medium — round-trip delay + user frustration |
| **Circuitous path** | Task Limit Exceeded | Agent took N steps when AGENTS.md should have provided a direct path in M < N steps | Highest — compounds across every extra step |
| **Repetition loop** | Task Limit Exceeded | Agent repeated the same action/search pattern without progress (AgentBench finding: >90% of TLE trajectories show Rouge-L ≥ 0.8 repetition in last 10 rounds) | Catastrophic — unbounded token burn |

### Deterministic pre-pass (before LLM judge)

The orchestrator should compute these metrics from the raw transcript BEFORE invoking the bug-identifier prompt. These are cheap, deterministic, and high-signal:

1. **Total tool calls** — count of Glob, Grep, Read, Edit, Write, Bash invocations
2. **Search-before-act ratio** — (Glob + Grep + Read calls) / (Edit + Write + Bash calls). High ratio on known-pattern tasks = AGENTS.md gap
3. **User correction count** — regex for "no", "not that", "use X instead", "stop", "don't" in user turns following agent actions
4. **Repetition detection** — flag when the same tool is called with similar arguments (cosine similarity > 0.8) within a sliding window of 5 calls. Per AgentBench J.2.4, this is the #1 cause of wasted tokens
5. **Token count per task segment** — if transcript has clear task boundaries, compute tokens per task

These metrics are passed to the bug-identifier as structured data alongside the raw transcript. The prompt uses them as evidence, not as the sole scoring mechanism.

### Output format

```json
[
  {
    "failure": "Searched for ticket naming convention",
    "category": "unnecessary_search",
    "responsible_section": "Boundaries",
    "evidence_from_transcript": "Line 42: Glob('docs/v0_5_*/**/*') — ticket format is specified in Boundaries section",
    "suggested_fix": "Add explicit example: clwdi-v{M}_{m}_{P}-{NNN}.md",
    "token_waste_estimate": 1200,
    "deterministic_signals": {"search_before_act_ratio": 3.2, "tool_calls_before_first_edit": 5}
  }
]
```

The prompt receives the current AGENTS.md alongside the transcript so it can identify what's missing vs. what's present but ignored.
