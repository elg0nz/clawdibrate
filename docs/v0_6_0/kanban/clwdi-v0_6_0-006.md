## Problem

`replace_section()` inserts `new_content` raw into a `re.sub` replacement string. Backslash sequences in `new_content` are interpreted as regex backreferences, silently corrupting AGENTS.md.

## Fix

Use a lambda:

```python
result = re.sub(pattern, lambda m: m.group(1) + new_content + "\n\n", agents_md, flags=re.DOTALL)
```

This carries forward into the new architecture — the implementer still outputs section content that gets spliced into AGENTS.md via `replace_section()`.
