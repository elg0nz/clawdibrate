## Problem

Clawdibrate's core value is reducing instruction file size without losing capabilities. Currently there's no token measurement — only word count ("Under 700 words") and line-level edit distance. No way to track whether calibration runs are actually shrinking token usage.

## Deliverable

1. **`clawdibrate/tokens.py`** — tiktoken-based token counter using `cl100k_base` encoding (Claude-compatible).
   - `count_tokens(text: str) -> int`
   - `count_section_tokens(content: str) -> dict[str, int]` — per-section breakdown
   - `count_file_tokens(path: Path) -> dict` — whole file + per-section

2. **Add `tiktoken` to `pyproject.toml` dependencies.**

3. **Integrate into calibration loop** (`orchestrator.py`):
   - Compute token counts before and after each calibration run
   - Log token delta per section and total in calibration summary
   - Persist token counts in `scores.jsonl` and `reflections.jsonl`

4. **`--token-budget` CLI flag** (default: current file token count = "never grow"):
   - Reject implementer changes that push total tokens above budget
   - Print warning when approaching budget (>90%)

5. **Calibration summary output:**
   ```
   Tokens: 1,247 → 1,198 (-49, -3.9%) | budget: 1,300
   ```
