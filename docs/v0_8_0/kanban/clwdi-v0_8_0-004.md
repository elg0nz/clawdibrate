## Problem

No quick way to see token usage without running a full calibration. Users need visibility into which sections are heaviest and whether the file is growing or shrinking.

## Deliverable

1. **`src/skills/tokens/SKILL.md`** — standalone skill that reads instruction file and prints:
   - Total token count and budget status
   - Per-section token breakdown, sorted by size (heaviest first)
   - Percentage of total per section
   - Trend arrows vs last calibration run (from scores.jsonl)

2. **Example output:**
   ```
   AGENTS.md token budget: 1,198 / 1,300 (92%)

   Section                  Tokens    %    Trend
   ─────────────────────────────────────────────
   Boundaries                 312   26%    ↓ -15
   Known Gotchas              198   17%    → 0
   Setup                      156   13%    ↓ -8
   Skills                     142   12%    → 0
   Bootstrap Transcript Cal   128   11%    ↑ +12
   Tuning Rules                98    8%    ↓ -22
   Commands                    64    5%    → 0
   Score Tracking              52    4%    → 0
   Identity                    28    2%    → 0
   References                  20    2%    → 0
   ```

3. **Register skill** via `npx skills add ./src/skills --agent <detected-agents> --skill '*' -y`.
