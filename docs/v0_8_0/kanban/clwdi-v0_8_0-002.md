## Problem

When token count exceeds budget, the user has no guidance on what to compress. The system should suggest specific compressions — redundant phrases, verbose patterns, prose that could be CLI commands.

## Deliverable

1. **Compression patterns library** — common bloat patterns in instruction files:
   - Redundant preamble ("In order to", "It is important to note that")
   - Prose descriptions of CLI commands (should be the command itself)
   - Duplicate information across sections
   - Verbose bullet points that could be terse

2. **Agent-powered compression pass** — new stage in calibration loop (optional, triggered when over budget):
   - Feed over-budget sections to LLM with prompt: "Compress this instruction section without losing any capability. Remove redundancy, prefer commands over prose, cut filler."
   - Measure before/after token count with tiktoken
   - Only accept compressions that preserve all actionable content

3. **`--compress` CLI flag** — run compression advisor standalone:
   ```bash
   python -m clawdibrate --compress
   ```

4. **Report per-suggestion token savings:**
   ```
   Suggestion: Boundaries section (312 → 248 tokens, -64)
     - Removed 3 redundant phrases
     - Converted 2 prose descriptions to CLI commands
   ```
