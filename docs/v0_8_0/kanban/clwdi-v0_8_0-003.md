## Problem

The existing section skills suggestion (`_suggest_section_skills`) uses score and git churn to recommend extraction. It doesn't consider token weight — a 50-token section with low score is less impactful to extract than a 300-token section.

## Deliverable

1. **Add token counts to skill suggestions.** `_suggest_section_skills()` now includes per-section token count in its output:
   ```
   [score=0.55, churn=4, tokens=287] 'Known Gotchas'
     → create src/skills/known-gotchas/SKILL.md
     → estimated savings: ~230 tokens (replace with 1-line pointer)
   ```

2. **Token-based extraction threshold.** Suggest extraction when a section exceeds a token threshold (default: 200 tokens), regardless of score/churn. Large sections are inherently good extraction candidates.

3. **Calculate token savings.** Estimate savings as `section_tokens - pointer_tokens` where pointer is ~15 tokens (`See /clawdbrt:<skill-name> for detailed guidance.`).

4. **Rank suggestions by token savings** (highest first) instead of alphabetical.
