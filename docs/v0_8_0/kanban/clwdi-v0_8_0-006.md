## Problem

The calibration loop (`orchestrator.py`) runs sequentially: one transcript at a time, one section fix at a time. Each stage (bug-identifier, judge, implementer) shells out to a single agent CLI call. This is slow — a run with 5 transcripts and 3 sections to fix makes ~15+ sequential agent calls.

## Deliverable

Replace sequential agent calls with parallel `claude -p "..." --model haiku` workers:

1. **Parallel transcript processing.** Each transcript gets its own worker for bug-identification. Fan out N transcripts → N parallel `claude -p` calls → collect failures.

2. **Parallel judge calls.** Each failure gets its own worker for scoring. Fan out M failures → M parallel judge calls → collect verdicts.

3. **Parallel section fixes.** Independent sections get parallel implementer workers. Only serialize when two failures target the same section.

4. **`--workers N` CLI flag.** Control parallelism (default: 4). `--workers 1` = current sequential behavior.

5. **`--model` CLI flag.** Override agent model for workers (default: haiku for speed, opus for quality).

6. **Worker invocation.** Each worker is `claude -p "{prompt}" --model {model} --dangerously-skip-permissions`. Prompt includes system prompt content inline. Workers write results to temp files, orchestrator collects.

7. **Also applies to `/clawdbrt:loop` skill.** The skill should use the same parallel worker infrastructure, not just the Python orchestrator.

## Architecture

```
orchestrator.py
  ├── fan-out: N transcript workers (bug-identifier)
  │     └── claude -p "..." --model haiku
  ├── collect failures
  ├── fan-out: M failure workers (judge)
  │     └── claude -p "..." --model haiku
  ├── collect verdicts
  ├── fan-out: K section workers (implementer)
  │     └── claude -p "..." --model haiku
  └── collect, validate, merge, commit
```

## Key Constraints

- Workers must not modify AGENTS.md directly — they return content, orchestrator merges
- Temp files for worker I/O: `/tmp/clawdibrate-worker-{uuid}.json`
- Workers inherit `CLAWDIBRATE_AGENT_CMD` if set, otherwise use `claude -p`
- Backpressure: if a worker times out (default 120s), skip that transcript/failure
